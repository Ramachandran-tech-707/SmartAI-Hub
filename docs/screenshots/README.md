## 🎯 How to Add Screenshots

1. **Capture Screenshots**: Take clear, high-quality screenshots of each feature
2. **Optimize**: Compress images without losing quality (tools: TinyPNG, ImageOptim)
3. **Name Properly**: Use descriptive, lowercase names with hyphens
4. **Add to Directory**: Save in this folder
5. **Update Documentation**: Add reference to main README.md

### Screenshot Guidelines

- **Resolution**: 1920x1080 or higher for high DPI displays
- **Format**: PNG (recommended) or WebP
- **Size**: Keep under 1MB per image
- **Content**: Show real data/examples where possible
- **Branding**: Include SmartAI-Hub interface elements

### Naming Convention

```
[feature]-[action/state].png
or
[tool]-[component].png

Examples:
- dashboard-home.png
- chatbot-conversation.png
- generator-form.png
- extractor-results.png
```

## 🖼️ Screenshot Checklist

When adding new screenshots, ensure:

- [ ] Screenshot is clear and readable
- [ ] Text is legible (no blurry text)
- [ ] No sensitive data visible
- [ ] Background is appropriate
- [ ] Consistent with other screenshots
- [ ] File is optimized for web
- [ ] Filename follows naming convention
- [ ] README is updated with reference

## 📝 Using Screenshots in Documentation

In README.md files, reference screenshots like:

```markdown
![Feature Description](./docs/screenshots/feature-name.png)
*Caption describing what's shown in the screenshot*
```

---

## 💡 Tips for Great Screenshots

1. **Clean Desktop** - Remove unnecessary icons/windows
2. **Light Theme** - Use light themes for consistency
3. **Cursor Hidden** - Hide mouse cursor in screenshots
4. **Highlight Features** - Use boxes/arrows to highlight important elements
5. **Show States** - Capture different states (loading, success, error)
6. **Include Text** - Show UI labels and button text clearly
7. **Consistent Sizing** - Keep dimensions consistent across screenshots
